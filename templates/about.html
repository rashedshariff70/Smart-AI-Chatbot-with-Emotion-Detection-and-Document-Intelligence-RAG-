About NeuralMind AI

A full-stack AI platform combining emotion intelligence, retrieval-augmented generation, and conversational AI — all powered by Groq's ultra-fast inference.

The Three Stages

How it works

Each stage is an independent AI pipeline sharing the same Qwen3-32B backbone with tailored system prompts.

1
Stage 1

Emotion Detection Engine

Takes raw text input, sends it to Qwen3-32B with a structured JSON-output system prompt, and parses the response into a rich emotion profile: 7 emotion scores (joy, sadness, anger, fear, surprise, disgust, neutral), an overall sentiment label, intensity level, and an empathetic one-line insight.

Zero-shot prompting Structured JSON output 7 emotion classes Confidence scoring
2
Stage 2

RAG Document Intelligence

Users upload a PDF, DOCX, or plain TXT file. The backend extracts the full text (PyPDF2 / python-docx), stores it per-session, then injects it as context into every query. The model is strictly grounded to the document — it won't hallucinate content that isn't there.

PyPDF2 · python-docx Session-scoped RAG Context-grounded answers 15K token doc window
3
Stage 3

Project Q&A Chatbot

A multi-turn chatbot that knows the NeuralMind project inside out — tech stack, architecture, API key format, deployment options, and more. Maintains a rolling 10-turn conversation history per session for coherent dialogue.

Multi-turn history Knowledge-scoped system prompt Session memory

Configuration

API key format

The project reads both the key and model name from a single .env variable using an inline comment convention:

# .env
GROQ_API_KEY_QWEN=gsk_XXXXXXXXXXXXXXXX#qwen/qwen3-32b_emotion_llm

The app splits on # — everything before is the API key, everything after is the model ID passed to client.chat.completions.create(model=...).
